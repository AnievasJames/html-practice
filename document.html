I. FUNCTIONAL REQUIREMENTS
Name Description Version
Functional Requirements
View Property Listings This function allows users to browse all available land properties in a specific town. The system displays a catalog of properties with basic information including location, price, and property size. Users can scroll through the listings to find properties of interest. Version 1.0
View Property Details This function allows users to view detailed information about a specific property. When a user selects a property from the listings, the system displays comprehensive details including:
  • Property images
  • Full description
  • Price
  • Location/address
  • Land size/dimensions
  • Contact information of seller/owner
 Version 1.0
Compare Properties This function allows users to compare two land properties side-by-side. Users can select two properties from the listings and view their details simultaneously to identify differences and similarities. The comparison includes:
  • Prices
  • Locations
  • Land sizes
  • Property features
  • Seller contact information
 Version 1.0
Search Properties This function allows users to search for properties based on specific criteria such as:
  • Location
  • Price range
  • Land size
The system filters the property listings based on the search parameters entered by the user. 		Version 1.0
Contact Seller This function allows users to view and access the contact information of property sellers/owners. The system displays:
  • Seller name
  • Phone number
  • Email address (if available)
Users can use this information to directly communicate with the seller regarding the property. 		Version 1.0

  1. View Property Listings

Relationships:

  • User can View Property Listings
  • User can View Property Details

III. USE CASE NARRATIVES

Use Case 1: View Property Listings

Actor Actions System Responses
1. User accesses the website 2. System displays the home page with property listings
3. System loads all available properties from database
4. System displays properties in a grid/list format showing basic information (image, price, location)
5. User scrolls through the listings
6. User clicks on a property for more details 7. System redirects to Property Details page (see View Property Details use case)

Alternative Scenarios

Actor Actions System Responses
3. System loads all available properties from database
4. No properties found in database
5. System displays message "No properties available at the moment"

Use Case 2: View Property Details

Actor Actions System Responses
1. User clicks on a property from listings 2. System retrieves property details from database
3. System displays Property Details page showing:
• Property images
• Price
• Description
• Location
• Land size
• Seller contact information
4. User views property images
5. User views property information
6. User clicks "Back to Listings" button 7. System returns to Property Listings page

Alternative Scenarios

Actor Actions System Responses
4. User clicks "Contact Seller" button 5. System highlights or displays seller contact information
6. User clicks "Compare" button 7. System adds property to comparison (see Compare Properties use case)

Use Case 3: Compare Properties

Actor Actions System Responses
1. User views Property Details page
2. User clicks "Add to Compare" button 3. System adds property to comparison list
4. System displays notification "Property added to comparison (1/2)"
5. User returns to Property Listings 6. System displays Property Listings with comparison indicator
7. User selects another property 8. System displays Property Details page
9. User clicks "Add to Compare" button 10. System adds second property to comparison
11. System displays notification "Property added to comparison (2/2)"
12. User clicks "Compare Now" button 13. System displays Comparison page with both properties side-by-side
14. System shows comparative information:
• Property images
• Prices
• Locations
• Land sizes
• Descriptions
• Seller contacts
15. User views comparison
16. User clicks "Clear Comparison" or selects a property 17. System removes properties from comparison or redirects to selected property

Alternative Scenarios

Actor Actions System Responses
9. User clicks "Add to Compare" button 10. System checks comparison list
11. Comparison list already has 2 properties
12. System displays message "Comparison full. Please remove a property first"
13. System displays option to remove properties from comparison

Use Case 4: Search Properties

Actor Actions System Responses
1. User accesses the website 2. System displays home page with search bar
3. User enters search criteria (location, price range, or land size)
4. User clicks "Search" button 5. System processes search parameters
6. System queries database for matching properties
7. System displays filtered results matching search criteria
8. System shows number of results found
9. User views search results
10. User clicks on a property 11. System displays Property Details page (see View Property Details use case)

Alternative Scenarios

Actor Actions System Responses
4. User clicks "Search" button 5. System processes search parameters
6. System queries database for matching properties
7. No properties match the search criteria
8. System displays message "No properties found matching your search. Please try different criteria"
9. User clicks "Clear Search" or "View All" 10. System displays all properties (see View Property Listings use case)

Use Case 5: Contact Seller

Actor Actions System Responses
1. User views Property Details page 2. System displays property information including seller contact section
3. User scrolls to "Contact Seller" section 4. System displays seller contact information:
• Seller name
• Phone number
• Email address (if available)
5. User copies phone number or email
6. User uses their own device to contact seller via phone or email

Alternative Scenarios

Actor Actions System Responses
3. User scrolls to "Contact Seller" section 4. Contact information is not available
5. System displays message "Contact information unavailable for this property"
